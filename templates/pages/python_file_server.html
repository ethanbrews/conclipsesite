Usage

The server will create a config.json file in the current working directory. This contains settings for where files should be saved, etc.

A web server will need to interact with the functions defined in interface.py.

Functions
add_pack(in_path, pack_id, pack_version, pack_version_readable, pack_author)
    str in_path: The path of the updated modpack
    str pack_id: The unique id of the modpack
    float pack_version: The version to be given to the new pack update
    str/None pack_version_readable: The version name to be displayed to users
    str pack_author: Author of the update. Separate multiple authors with commas
    Returns: bool Whether pack was updated successfully

update_pack(in_path, pack_id, pack_version, pack_version_readable, pack_author)
    str in_path: The path of the modpack
    str pack_id: The unique id of the modpack
    float pack_version: The current version of the pack
    str/None pack_version_readable: The version name to be displayed to users
    str pack_author: Author of the pack. Separate multiple authors with commas
    Returns: bool Whether pack was added successfully


get_update(pack_id, current_version, target_version)
    str pack_id: The unique id of the modpack
    float pack_version: The version the user has
    float pack_version: The version the user wants
    Returns: bytes Bytes data of zip file containing the update
